V5 下一步实施计划：Validation-Calibrated State-Dependent RCVaR

目标： 把当前 V3/V4 中“默认 $C_{\text{manual}}$ 在 SP100 上不稳定”的问题，转化为一个规范的、可写论文的 V5 协议：

$$ \boxed{ \text{用 validation 选择 } \kappa_{\max} + \text{控制权重集中与换手} + \text{在 test 上只评价一次} } $$
核心原则： 以后不是“每个市场看 test 结果后手工调 $\kappa_{\max}$”，而是每个市场都执行同一套 validation calibration 规则。

不能做： $$ \text{看 2020--2024 test，哪个 }\kappa_{\max}\text{ 好就选哪个。} $$ 应该做： $$ \kappa_{\max}^* = \arg\min_{\kappa_{\max}\in\mathcal K} J_{\text{val}}(\kappa_{\max}), \quad \text{然后在 test 固定 }\kappa_{\max}^*. $$

1. 先解决 A_no_kappa 与 Historical_CVaR 的关系

1.1 理论上它们是不是一样？

如果二者都使用同一个历史窗口、同一个损失定义、同一个交易成本口径、同一个优化器和同一组约束，那么：

$$ A_{\text{no kappa}}: \kappa=1 $$

对应的 RCVaR 风险包络为：

$$ q_i\le \frac{1}{\alpha B}. $$

这正好退化为普通经验 CVaR。因此：

$$ \boxed{ A_{\text{no kappa}} \equiv \text{Historical CVaR} } $$

也就是说，理论上 A_no_kappa 本质上就是历史经验 CVaR 优化。

1.2 你当前文件显示了什么？

根据当前上传的权重文件：

weights_A.csv
weights_HistoricalCVaR.csv

两者形状相同，且所有权重完全一致：

$$ \max_{t,i}|w^{A}_{i,t}-w^{Hist}_{i,t}|=0. $$
这意味着： 至少从权重文件看，A_no_kappa 和 Historical_CVaR 是同一个组合权重序列。

1.3 为什么之前结果会不一样？

如果权重完全一样，但 summary table 中 A_no_kappa 和 Historical_CVaR 的 CVaR、MDD、Sharpe 不一样，那么只能说明评价口径或文件读取出了问题。常见原因包括：

可能原因说明处理
评价收益序列不同 A 用一个 test returns，Historical 用另一个 returns 或 benchmark-aligned returns。 统一读取同一个 returns_test.csv。
交易成本口径不同 A 可能使用了调仓成本，Historical 可能没有，或反过来。 统一 net_return = gross_return - cost。
rolling 文件缺失 当前没有 rolling_HistoricalCVaR.csv，所以无法验证 Historical 的日度收益路径。 必须重新生成 Historical 的 rolling 文件。
baseline 权重误读 诊断代码可能把 Historical 的权重读成了 A 的权重。 在代码里打印文件路径、hash 和前几行权重。
summary table 混入旧结果 Historical_CVaR 可能来自旧版本 V3 table，不是当前权重重新计算出的结果。 清空输出目录后重新汇总。

1.5 论文命名建议

如果审计确认 A_no_kappa = Historical_CVaR，则论文中不要同时作为两个不同策略报告。建议统一改名：

$$ \boxed{ A:\ \text{Historical CVaR }(\kappa=1) } $$

然后 B、C、D 分别作为 fixed robust、state-dependent robust、state-action-dependent robust。

2. V5 总体目标

V5 不要马上做复杂 RL，也不要立刻做 asset-aware $\kappa_i(s)$。先做一个最小但规范的版本：

$$ \boxed{ C_{\text{calibrated}} = \text{validation-calibrated state-dependent RCVaR} } $$

核心公式：

$$ \kappa_t^{raw} = 1+ \kappa_{\max} \sigma(\beta^\top s_t). $$

可选平滑：

$$ \bar{\kappa}_t = \rho\bar{\kappa}_{t-1} + (1-\rho)\kappa_t^{raw}. $$

最终优化：

$$ w_t^* = \arg\min_{w\in\Delta^N} RCVaR_{\alpha,\bar{\kappa}_t} \left( \{-w^\top r_\tau\}_{\tau=t-M}^{t-1} \right) + c\|w-w_{t-1}\|_1 + \lambda_h\sum_i w_i^2. $$

可选权重上限：

$$ 0\le w_i\le u,\qquad \sum_iw_i=1. $$

3. 数据集与时间划分协议

后面不只是 SP100，所以要统一协议。每个市场都按同一套流程跑。

阶段时间用途
Train2010--2017下载、清洗、状态标准化参数、初始验证。
Validation2018--2019选择 $\kappa_{\max}$、可选 $u$、可选 $\rho$。
Test2020--2024最终样本外评价，只能跑一次汇总，不用于调参。
如果某个市场数据起点不同，可以保持 test 期不变；train/validation 尽量保持一致。所有市场必须在报告里写清楚有效样本期。

4. 每个市场是否都要不同 $\kappa_{\max}$？

4.1 正确答案

可以不同，但必须由同一套 validation 规则自动选择。

$$ \boxed{ \kappa_{\max}\text{ 是模型超参数，可以按资产池校准。} } $$

原因是不同资产池的尾部结构不同：

市场/资产池特征$\kappa_{\max}$ 可能表现
ETF10低维、多资产分散$\kappa_{\max}=1$ 可能稳定。
ETF20更分散，尾部被平滑$\kappa$ 边际作用可能弱。
SP30中等股票池状态依赖 $\kappa(s)$ 可能有效。
SP100高维股票池，个股噪声大需要校准 $\kappa_{\max}$ 并控制权重集中。

4.2 不正确做法

for each dataset:
    在 test 上跑很多 kappa_max
    选择 test CVaR 最低的值
    当作主结果

这是 test-set tuning，会导致 data snooping。

4.3 正确做法

for each dataset:
    在 validation 上搜索 kappa_max
    固定选出的 kappa_max
    在 test 上评价一次

论文可写：

For each universe, $\kappa_{\max}$ is selected on the validation period and fixed before out-of-sample testing. No test-period information is used for parameter selection.

5. Validation 目标函数设计

5.1 不建议只用 validation CVaR

如果只选：

$$ \arg\min CVaR^{val}_{5\%}, $$

可能选出换手很高、权重很集中的参数。V4 诊断已经显示 SP100 的问题正是高换手与集中。

5.2 推荐目标函数

$$ J_{\text{val}} = CVaR^{val}_{5\%} + \lambda_{turn}\overline{Turn}^{val} + \lambda_{hhi}\overline{HHI}^{val}. $$
项含义
$CVaR^{val}_{5\%}$验证期尾部风险。
$\overline{Turn}^{val}$验证期平均换手率。
$\overline{HHI}^{val}$验证期平均权重集中度。

5.3 简化版目标

如果你想先简单执行：

$$ J_{\text{val}} = CVaR^{val}_{5\%} + 0.001\overline{Turn}^{val}. $$

更稳健版本：

$$ J_{\text{val}} = CVaR^{val}_{5\%} + 0.001\overline{Turn}^{val} + 0.01\overline{HHI}^{val}. $$
这些系数可以先作为工程默认值，但必须在所有市场统一使用。不要每个市场单独调 $\lambda$。

6. V5 模型集合

每个市场至少跑以下模型：

模型名定义论文角色
A_Historical_CVaR $\kappa=1$ 普通历史 CVaR baseline。
B_fixed_kappa $\kappa=2$ 固定鲁棒 baseline。
C_default $\kappa_{\max}=1.0$ 未校准状态依赖模型。
C_calibrated $\kappa_{\max}$ 由 validation 选择 V5 主模型。
C_cap C_calibrated + $w_i\le u$ 控制集中度的增强模型。
C_smooth C_calibrated + $\kappa$ 平滑 控制换手的增强模型。
C_stable C_calibrated + cap + smoothing 最终稳定版本候选。

10. 成功标准

10.1 最低成功标准

每个市场中，C_calibrated 至少应满足：

$$ CVaR(C_{\text{calibrated}}) < CVaR(A_{\text{Historical}}) $$

或者如果不能满足，至少要说明原因。

10.2 强成功标准

$$ CVaR(C_{\text{stable}}) < \min\{ CVaR(A_{\text{Historical}}), CVaR(B_{\text{fixed}}) \}. $$

10.3 稳定性标准

不能只看 CVaR，还要看：

指标目标
平均换手不应明显高于 A/B，例如不超过 B 的 1.5 倍。
有效持仓数SP100 不应长期低于 50。
最大单权重建议不超过 8% 或 10%。
MDD不应显著高于 A/B。
Bootstrap$P(CVaR_C < CVaR_A)$ 越高越好，最好超过 70%。

13. 论文写法建议

13.1 方法部分

不要说“我们手动选择每个市场最好的 $\kappa_{\max}$”。应该写：

The amplitude parameter $\kappa_{\max}$ is treated as a hyperparameter and selected on the validation period for each asset universe using the same pre-specified selection rule. The selected value is then fixed throughout the test period.

13.2 实证部分

可以分成三张主表：

表内容
Table 1单市场主结果，例如 ETF10 或 SP100。
Table 2跨市场 CVaR 对比。
Table 3Validation-selected 参数表。
Table 4权重集中度与换手诊断。
Table 5Bootstrap 显著性。

13.3 Discussion

SP100 的 V4 诊断可以写成方法边界：

In high-dimensional stock universes, a default state-dependent ambiguity budget may induce excessive turnover and concentrated portfolios. This motivates the validation-calibrated and stability-regularized version of the proposed model.

14. 最终结论（计划阶段）

下一步不是“每个市场随便调 $\kappa_{\max}$”，而是建立一个固定、可复现、没有 test leakage 的校准协议：

$$ \boxed{ \text{validation 选择参数} \rightarrow \text{test 固定评价} \rightarrow \text{跨市场统一汇总} } $$
如果 V5 跑通，你的论文主线会从“默认 $\kappa(s)$ 是否总是有效”升级为：

$$ \boxed{ \text{状态依赖鲁棒 CVaR 是有效的，但高维市场需要 validation calibration 和稳定性控制。} } $$
