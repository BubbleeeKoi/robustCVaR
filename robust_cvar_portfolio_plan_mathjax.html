1. 项目最终目标

本项目的最终目标是提出一种面向真实股票市场的 State-Action-Dependent Robust CVaR Portfolio Learning 方法，用于动态投资组合优化。

论文暂定题目可以是：

Regime-Adaptive State-Action-Dependent Robust CVaR Portfolio Learning for Dynamic Stock Allocation

或：State-Action-Dependent Robust CVaR Reinforcement Learning for Dynamic Portfolio Optimization

本文的核心目标不是追求最高收益，而是在保持合理收益的同时，显著降低：

\[ CVaR_{5\%},\quad VaR_{5\%},\quad Maximum\ Drawdown,\quad Crisis\ Period\ Loss. \]

首要评价指标为：

\[ \boxed{\text{Out-of-sample }CVaR_{5\%}\text{ of net portfolio loss}.} \]

本文基于 Ni & Lai 的 robust CVaR-RL 思想，但不是继续做 GridWorld 复现，也不是把 GridWorld 直接换成股票数据。原文研究的是有限 RMDP 中的 robust CVaR 与 NCVaR value iteration；本文要迁移的是 robust CVaR、fixed ambiguity、state-action-dependent ambiguity budget 这三条方法主线。

2. 为什么要修改执行计划？

前面的总体方向仍然正确：

\[ \kappa(s,w)+RCVaR_{\alpha,\kappa}+Portfolio\ Learning. \]

但执行顺序要修改。不要一开始直接做 PPO / Actor-Critic，因为金融 RL 噪声大、训练不稳定、结果难解释。如果 robust CVaR 风险层本身还没验证，就直接接 PPO，后面出问题很难定位。

修改后的路线应为：

\[ \boxed{\text{数据与指标}\rightarrow\text{robust CVaR layer}\rightarrow\text{非 RL rolling portfolio}\rightarrow\text{PortfolioEnv}\rightarrow\text{PPO}\rightarrow\text{CVaR-PPO}\rightarrow\text{Robust CVaR-PPO}\rightarrow\text{State-action robust CVaR-PPO}.} \]

核心修改：PPO 后置，robust CVaR layer 和 rolling portfolio 前置。先把金融风险层验证出来，再进入 RL。

3. 原文方法与本文方法的关系

原文核心模型为：

\[ \min_{\pi}\max_{\tilde P\in\mathcal P} CVaR_\alpha\left(\sum_{t=0}^{\infty}\gamma^t C(x_t,a_t)\right). \]

本文将其改写为金融动态组合问题：

\[ \min_{\pi_\theta}\max_{\tilde P\in\mathcal P_{\kappa(s_t,w_t)}}CVaR_\alpha\left(\sum_{t=0}^{T-1}\gamma^t L_{t+1}(s_t,w_t)\right). \]

严谨表述：原文是 robust risk-sensitive RL / RMDP 理论框架，但实现是 value iteration，不是 PPO。本文的 PPO / actor-critic 是为了适配真实金融连续动作问题而做的扩展。

4. 核心创新点

4.1 金融状态-动作依赖 ambiguity budget

本文提出：

\[ \kappa_t=\kappa(s_t,w_t). \]

含义是：市场越不稳定、组合动作越激进，模型不确定预算越大。例如高波动、高回撤、高相关、高集中度、高换手都应使 \(\kappa_t\) 增大。

4.2 Robust CVaR risk layer

本文不把 CVaR 简单作为 reward penalty，而是用对偶形式构造风险层：

\[ RCVaR_{\alpha,\kappa}(Z)=\max_{q\in\mathcal Q_{\alpha,\kappa}}\sum_{i=1}^{B}q_iZ_i, \] \[ \mathcal Q_{\alpha,\kappa}=\left\{q\in\mathbb R_+^B:\sum_{i=1}^{B}q_i=1,\ 0\le q_i\le \frac{\kappa_i}{\alpha B}\right\}. \]

当 \(\kappa_i=1\) 时退化为普通 CVaR；当 \(\kappa_i=K\) 时变成 fixed robust CVaR；当 \(\kappa_i=\kappa(s_i,w_i)\) 时就是本文的状态-动作依赖鲁棒 CVaR。

4.3 渐进式实现

先做非 RL 的 rolling robust CVaR portfolio，验证风险层有效，再接入 PPO / actor-critic。这样更稳，也更容易写成论文。

5. 数学模型

5.1 股票收益与组合权重

设市场中有 \(N\) 只股票，第 \(t\) 期收益向量为：

\[ r_t\in\mathbb R^N. \]

组合权重为：

\[ w_t\in\Delta^N, \] \[ \Delta^N=\left\{w\in\mathbb R^N:w_i\ge0,\ \mathbf 1^\top w=1\right\}. \]

5.2 市场状态

\[ s_t=\left(R_{t-L:t},z_t,w_{t-1}\right), \] \[ R_{t-L:t}=[r_{t-L},r_{t-L+1},\dots,r_t]\in\mathbb R^{L\times N}. \]

其中 \(z_t\) 包括市场波动率、平均相关性、市场回撤、近期尾部亏损强度和横截面离散度：

\[ z_t=(Vol_t,Corr_t,DD_t,Tail_t,Disp_t). \]

5.3 净收益与净损失

组合毛收益为：

\[ R_{t+1}^{p}=w_t^\top r_{t+1}. \]

换手率为：

\[ TO_t=\|w_t-w_{t-1}\|_1. \]

设交易成本率为 \(c>0\)，组合净收益为：

\[ R_{t+1}^{net}=w_t^\top r_{t+1}-c\|w_t-w_{t-1}\|_1. \]

净损失定义为：

\[ \boxed{L_{t+1}(s_t,w_t)=-w_t^\top r_{t+1}+c\|w_t-w_{t-1}\|_1.} \]

累计损失为：

\[ Z^\pi=\sum_{t=0}^{T-1}\gamma^tL_{t+1}(s_t,w_t). \]

金融回测中可以取 \(\gamma=1\)。

5.4 普通 CVaR

给定损失样本 \(Z_1,\dots,Z_B\)，普通经验 CVaR 可写为：

\[ CVaR_\alpha(Z)=\max_{q\in\mathcal Q_\alpha}\sum_{i=1}^{B}q_iZ_i, \] \[ \mathcal Q_\alpha=\left\{q\in\mathbb R_+^B:\sum_{i=1}^{B}q_i=1,\ 0\le q_i\le\frac{1}{\alpha B}\right\}. \]

5.5 状态-动作依赖 Robust CVaR

\[ \boxed{RCVaR_{\alpha,\kappa}(Z)=\max_{q\in\mathcal Q_{\alpha,\kappa}}\sum_{i=1}^{B}q_iZ_i} \] \[ \boxed{\mathcal Q_{\alpha,\kappa}=\left\{q\in\mathbb R_+^B:\sum_{i=1}^{B}q_i=1,\ 0\le q_i\le\frac{\kappa_i}{\alpha B}\right\}.} \]

其中：

\[ \kappa_i=\kappa(s_i,w_i). \]

这个形式表示：高不确定市场状态或高风险动作对应的损失样本，可以在 worst-case tail distribution 中获得更高权重。

6. \(\kappa(s,w)\) 设计

6.1 第一版简化设计

第一版只使用三个变量：

\[ Vol_t,\quad DD_t,\quad Conc_t. \]

组合集中度：

\[ Conc_t=\sum_{i=1}^{N}w_{i,t}^2. \]

定义：

\[ \kappa(s_t,w_t)=1+\kappa_{\max}\sigma\left(\beta_1\widetilde{Vol}_t+\beta_2\widetilde{DD}_t+\beta_3\widetilde{Conc}_t\right), \] \[ \sigma(x)=\frac{1}{1+e^{-x}}. \]

因此：

\[ \kappa(s_t,w_t)\in[1,1+\kappa_{\max}]. \]

第一版取 \(\kappa_{\max}=1\)，因此 \(\kappa(s_t,w_t)\in[1,2]\)。

6.2 完整版设计

\[ \kappa(s_t,w_t)=1+\kappa_{\max}\sigma\left(\beta_0+\beta_1\widetilde{Vol}_t+\beta_2\widetilde{Corr}_t+\beta_3\widetilde{DD}_t+\beta_4\widetilde{Tail}_t+\beta_5\widetilde{Conc}_t+\beta_6\widetilde{Turn}_t\right), \] \[ Turn_t=\|w_t-w_{t-1}\|_1. \]

所有特征使用 rolling standardization：

\[ \widetilde{x}_t=\frac{x_t-\mu_{t,L}}{\sigma_{t,L}+\epsilon}. \]

7. 退化关系

7.1 退化为普通 CVaR

若 \(\kappa_i=1\)，则：

\[ RCVaR_{\alpha,\kappa}(Z)=CVaR_\alpha(Z). \]

7.2 退化为 fixed robust CVaR

若 \(\kappa_i=K\)，则：

\[ q_i\le\frac{K}{\alpha B}. \]

等价于更低置信水平：

\[ \alpha'=\frac{\alpha}{K}, \] \[ RCVaR_{\alpha,K}(Z)=CVaR_{\alpha/K}(Z). \]

7.3 状态-动作依赖 robust CVaR

若 \(\kappa_i=\kappa(s_i,w_i)\)，则每个样本的尾部权重上界随市场状态和组合动作动态变化。

10.3 Robust CVaR layer

robust_cvar_layer.py
robust_cvar(losses, kappa, alpha=0.05)

计算：

\[ RCVaR_{\alpha,\kappa}(Z)=\max_q\sum_iq_iZ_i, \quad 0\le q_i\le\frac{\kappa_i}{\alpha B},\quad \sum_iq_i=1. \]

实现：按损失从大到小排序，从最大损失开始按上界 \(u_i=\kappa_i/(\alpha B)\) 分配权重，直到总权重为 1。

单元测试：

  • \(\kappa_i=1\) 时等于普通 CVaR；
  • \(\kappa_i=K\) 时结果不小于普通 CVaR；
  • 高损失样本的 \(\kappa_i\) 增大时，RCVaR 增大。

11. 阶段 2：非 RL rolling robust CVaR portfolio

这是关键修改。先用非 RL 方法验证核心风险模型。

在每个调仓时点 \(t\)，用过去 \(M\) 天收益作为估计窗口，求解：

\[ \min_{w\in\Delta^N} RCVaR_{\alpha,\kappa(s_t,w)}\left(\left\{-w^\top r_\tau+c\|w-w_{t-1}\|_1\right\}_{\tau=t-M}^{t-1}\right). \]

约束：

\[ w_i\ge0,\quad \sum_iw_i=1. \]

第一版调仓频率：monthly 或 weekly；不建议 daily。估计窗口建议 \(M=252\)，后续比较 \(M\in\{252,504\}\)。

13. 阶段 4：CVaR-PPO 与 fixed robust CVaR-PPO

13.1 CVaR-PPO

设置 \(\kappa=1\)，此时 \(RCVaR_{\alpha,\kappa}=CVaR_\alpha\)。目标：

\[ \min_\theta CVaR_\alpha(Z^\pi). \]

13.2 Fixed robust CVaR-PPO

设置 \(\kappa=K\)，例如 \(K=1.5\) 或 \(K=2\)。此时：

\[ RCVaR_{\alpha,K}(Z)=CVaR_{\alpha/K}(Z). \]

比较 PPO、CVaR-PPO 和 fixed robust CVaR-PPO，期望 fixed robust 版本在 CVaR 和 MDD 上优于 PPO。

14. 阶段 5：State-dependent 与 State-action-dependent robust CVaR-PPO

14.1 State-dependent robust CVaR-PPO

\[ \kappa_i=\kappa(s_i)=1+\kappa_{\max}\sigma(\beta_1\widetilde{Vol}_t+\beta_2\widetilde{DD}_t). \]

14.2 State-action-dependent robust CVaR-PPO

\[ \kappa_i=\kappa(s_i,w_i)=1+\kappa_{\max}\sigma(\beta_1\widetilde{Vol}_t+\beta_2\widetilde{DD}_t+\beta_3\widetilde{Conc}_t). \]

最终比较：

\[ \kappa=1,\quad \kappa=K,\quad \kappa(s),\quad \kappa(s,w). \]

期望最终模型在 \(CVaR_{5\%}\)、MDD、Crisis Loss 上更好。

15. 阶段 6：Distributional Actor-Critic 扩展

若 PPO 版本稳定，再实现 quantile critic：

\[ Z_\psi(s,w)=\{z_1,z_2,\dots,z_M\}. \]

actor 优化：

\[ \min_\theta RCVaR_{\alpha,\kappa(s,\pi_\theta(s))}\left(Z_\psi(s,\pi_\theta(s))\right). \]

该版本更贴近 CVaR 的分布尾部本质，可作为论文增强版或后续拓展。

19. 可行性判断

这个方向是可行的，但不能把目标写成“收益最高”。真正目标应写成：

\[ \boxed{\text{在保持合理收益的同时降低尾部风险、最大回撤和危机期损失。}} \]

真正的创新点在于：

\[ \boxed{\kappa(s,w)+RCVaR_{\alpha,\kappa}+dynamic\ portfolio\ learning.} \]

下一步立即执行：data pipeline + risk metrics + robust CVaR layer + rolling robust portfolio。不要先写复杂 PPO。先把 robust CVaR 金融风险层验证出来，这是整篇论文的基础。
