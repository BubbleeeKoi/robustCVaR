# 复现实验设计与代码核查

## 文件作用

本文件是中文复现设计书和代码核查报告，负责把论文方法逻辑、Figure 1 实验参数、代码模块职责、从零实现步骤、运行命令和复现边界系统梳理清楚。

## 1. 论文复现目标

论文《Robust Risk-Sensitive Reinforcement Learning with Conditional Value-at-Risk》的核心问题是：在转移概率存在模型不确定性时，不只最小化期望总代价，而是最小化折扣总代价的最坏情形 CVaR。

本项目只复现论文 Figure 1 的 GridWorld 实验。严格目标是：

1. 按论文公开的 RMDP/CVaR/NCVaR 逻辑实现可运行代码。
2. 用论文 PDF 中嵌入的 Figure 1 图像抽取障碍物和红色路径，作为外部校验。
3. 对论文未公开的数据和超参数明确标注假设，不伪装成完全恢复作者原始实验。

## 2. 论文方法逻辑

### 2.1 固定不确定预算

论文先考虑矩形 ambiguity set，目标为：

```text
min_policy max_transition CVaR_alpha(total discounted cost)
```

利用 CVaR 的对偶表示，可以把外层转移不确定性和 CVaR 内部的最坏分布合并为新的风险敏感问题。

对 Figure 1 的固定预算面板：

- Radon-Nikodym ambiguity：预算 `K=2` 等价于把 CVaR 置信水平改为 `alpha' = alpha / K`。
- KL ambiguity：论文通过 EVaR 对偶关系处理；Figure 1 报告 `alpha'_EVaR = 0.03`。项目现在把两个目的分开：主 Figure 1c 使用 calibrated 半径 `0.03` 复现红线路径；Section III.B 的公式半径作为独立诊断输出，不覆盖主图。

因此代码中 Figure 1b 不应重新发明一个独立算法，而应读取同一个 CVaR 值函数表面在 `alpha / 2 = 0.24` 处的策略；Figure 1c 则求 KL/EVaR robust Bellman 更新。

### 2.2 决策相关不确定预算

论文进一步讨论状态-动作相关的 RN 预算 `kappa(x,a)`。这时不能只用一个固定 `alpha / K` 代替，而要引入 NCVaR，并在增广状态 `(x, y)` 上做动态规划：

- `x` 是 GridWorld 状态。
- `y` 是当前 CVaR/NCVaR 置信水平。
- `V(x,y)` 是从状态 `x`、置信水平 `y` 出发的最优风险敏感代价。

论文 Algorithm 2 用有限个 `y` 采样点做线性插值，并要求对 `y V(x,y)` 插值，而不是直接对 `V(x,y)` 插值。当前代码的主求解器 `solve_robust_cvar_pwl` 正是围绕这个透视形式实现。

## 3. Figure 1 实验参数

从论文正文和图注可以直接恢复：

- GridWorld 大小：`64 x 53`
- 起点：`(60, 50)`
- 终点：`(60, 2)`
- 动作：east, south, west, north
- 选择动作成功概率：`0.95`
- 其他三个方向概率：`0.05 / 3`
- 障碍物数量：`80`
- 安全移动代价：`1`
- 障碍物碰撞代价：`40`
- Figure 1a-c 置信水平：`alpha = 0.48`
- RN 与 KL 预算：`K = 2`
- RN 等价置信水平：`alpha'_CVaR = 0.24`
- KL/EVaR 等价置信水平：`alpha'_EVaR = 0.03`
- 线性插值采样点：`21` 个
- Figure 1d 不确定预算范围：`kappa(x,a) in [1, 2]`

论文没有公开：

- 80 个障碍物的精确坐标。
- 折扣因子 `gamma`。
- 碰到障碍物之后是否终止，以及撞墙后的状态转移方式。
- 初始值函数、停止准则和作者原始代码细节。
- 21 点几何置信网格中的具体 `theta` 数值。
- Figure 1d 的完整 `kappa(x,a)` 向量。
- Figure 1 的色条归一化、线宽、坐标映射等绘图细节。

当前代码采用 `gamma=0.95`；障碍物来自 PDF 图像抽取；障碍物碰撞按终止状态处理；撞墙按留在原地且产生碰撞代价处理；`theta=2.067` 作为继承自相关 GridWorld 设置的项目假设；Figure 1d 的 `kappa(x,a)` 使用局部碰撞风险构造并显式标为假设。

## 4. 代码结构核查

### 4.1 `robust_cvar_repro/gridworld.py`

负责环境：

1. `GridWorldConfig` 固定论文公开参数。
2. `PAPER_FIGURE_OBSTACLES` 保存从 Figure 1 图像抽取的 80 个障碍物坐标。
3. `_build_transitions` 生成四方向随机转移，并合并重复后继状态概率。
4. `expected_costs` 保存每个状态动作的一步期望代价。
5. `terminal_mask` 把终点和障碍物设为终止状态。
6. `kappa_decision_dependent` 用局部障碍碰撞风险生成 `[1,2]` 范围内的可解释假设。

核查重点：

- 状态数应为 `64 * 53 = 3392`。
- 障碍物数量应为 `80`。
- 每个 `(state, action)` 的转移概率和应为 `1`。
- 起点和终点不能落在障碍物中。

### 4.2 `robust_cvar_repro/value_iteration.py`

负责算法：

1. `confidence_grid` 构造 21 点置信网格，包含 `0` 和 `1`；`theta=2.067` 是代码假设，不是 PDF 实验段落直接公开值。
2. `solve_risk_neutral_value_iteration` 用作 CVaR 表面初始化。
3. `solve_robust_cvar_pwl` 实现 Algorithm 2 风格的分段线性 CVaR/NCVaR Bellman 更新。
4. `_robust_continuation_pwl` 在每个后继分布上做最坏情形分配，本质是在 `y V(x,y)` 的分段斜率上优先分配尾部预算。
5. `interpolate_value_surface` 对 `y V(x,y)` 插值后除以目标 `y`。
6. `evar_confidence_from_kl_budget` / `kl_radius_from_kl_budget` 按 Section III.B 把 `kappa=2` 映射为等价 EVaR 置信水平与 KL 半径；`evar_kl_radius` 保留纯 EVaR 对偶用途。
7. `_robust_continuation_pwl` 不再把 `kappa>=1` 截成 1；对论文未定义的 `y*xi>1` 使用风险中性端点扩展。
8. `extract_path` 会沿最优风险分配更新增广风险状态，并投影到论文规定的 `[0,1]` 域。
9. `evaluation.py` 用随机 rollout 输出 VaR、CVaR、到达率、碰撞率和风险状态投影率。

已删除的冗余内容：

- 旧的非分段线性 CVaR 更新。
- 旧的单 alpha 近似 CVaR 求解器。
- 旧的视觉校准脚本入口。
- Python 缓存和旧近似/校准输出。

### 4.3 `robust_cvar_repro/render.py`

负责输出 2x2 图：

- `render_comparison` 输出项目自己的网格风格图。
- `render_paper_style_comparison` 输出更接近论文布局的 matplotlib 图。

绘图只用于展示，不作为算法真实性的依据。

### 4.4 `scripts/`

当前只保留三条复现主线：

1. `extract_paper_figure_data.py`：必要时先用 PyMuPDF 从 PDF 第 5 页抽出四张 Figure 1 面板图，再从图像中抽取障碍物和红色路径。
2. `run_strict_gridworld.py`：运行严格复现，生成四个面板的值函数、策略、路径和图片。
3. `compare_paths_to_paper.py`：把复现路径与 PDF 图像抽取路径做 mean nearest grid distance 比较。

## 5. 从零设计复现代码的步骤

### 步骤 1：恢复论文参数

先从 PDF 正文记录公开参数，再从 Figure 1 图像中抽取未公开的视觉数据。

代码任务：

1. 用 PyMuPDF 打开 PDF，读取第 5 页的 4 张嵌入图像。
2. 保存为 `outputs/pdf_images/page5_image*.png`。
3. 识别障碍物颜色，映射到 `64 x 53` 网格。
4. 识别红色路径像素，映射到网格路径。
5. 导出 CSV，供后续比较。

运行：

```bash
python scripts/extract_paper_figure_data.py
```

### 步骤 2：实现 GridWorld

按论文设置实现环境：

1. 坐标 `(x,y)` 与线性状态 `state = y * width + x` 互转。
2. 每个动作有四个尝试方向：目标方向概率 `0.95`，其他方向概率 `0.05/3`。
3. 出界行为论文未写明；当前代码按碰撞处理并留在原地。
4. 碰到障碍物的后续动力学论文未写明；当前代码按进入障碍终止状态处理，代价 `40`。
5. 正常移动代价 `1`。
6. 终点价值固定为 `0`。

### 步骤 3：构造置信水平网格

论文使用 21 个插值点，并说明 `y_{i+1}=theta y_i`。PDF 实验段落没有给出 `theta` 的具体值。代码采用：

```text
y_grid = [0, theta^-19, theta^-18, ..., theta^0]
theta = 2.067
```

这让小 `y` 区域更密集，适合尾部风险计算；但 `theta=2.067` 应在报告中写成项目假设或引用自相关设置，而不是写成本文 PDF 直接公开参数。

### 步骤 4：实现 CVaR/NCVaR Bellman 更新

对每个 `state`、`y` 和 `action`：

1. 取后继状态集合和基准概率。
2. 对每个后继状态读取旧的 `y_grid * V(next_state, y_grid)`。
3. 把各后继的分段斜率按从大到小排序。
4. 根据 RN 上界 `u <= kappa` 和总预算 `y` 分配最坏情形质量。
5. 得到 robust continuation，再加一步代价和折扣项。
6. 对动作取最小值，得到新 `V(state,y)` 和 greedy policy。

无不确定性时 `kappa=1`。决策相关面板使用 `kappa(x,a)` 矩阵。

### 步骤 5：实现 Figure 1 四个面板

1. Figure 1a：求 `kappa=1` 的 CVaR 表面，在 `alpha=0.48` 处读取值函数和策略。
2. Figure 1b：复用同一个 CVaR 表面，在 `alpha/2=0.24` 处读取值函数和策略。
3. Figure 1c：主四图采用 calibrated KL 半径 `0.03`，得到 97 个路径点并接近论文的 102 点红线。另按 Section III.B 计算 `kappa=2` 的公式半径约 `2.18`，保存为 `diagnostic_*`，用于审计而不进入主图。
4. Figure 1d：用假设的 `kappa(x,a) in [1,2]` 求 NCVaR 表面，在 `alpha=0.48` 处读取值函数和策略。对 `y*xi>1` 使用显式风险中性端点扩展，并记录轨迹投影率。

主运行入口：

```bash
python scripts/run_strict_gridworld.py
```

先做静态检查：

```bash

```

评价口径：

- Figure 1a 应最接近，因为它依赖的未公开信息最少。
- Figure 1b 和 1c 会受障碍物抽取误差、折扣因子和路径展示方式影响。
- Figure 1d 只能验证“决策相关不确定预算使策略更保守”的趋势，因为论文没有公开 `kappa(x,a)`。

## 6. 当前复现边界

本项目已经具备可审计复现流程，但当前结果不能称为 Figure 1 的合格严格复现。原因是论文 PDF 只给出图像和文字参数，没有给出完整实验数据；同时旧版 KL/EVaR 更新和部分路径提取方式会产生明显偏差。最终报告应同时提交：

- 严格复现图片。
- 四个面板的路径 CSV。
- 值函数/策略 `.npy`。
- 与 PDF 抽取路径的距离比较。
- `REPRO_STATUS.md` 中列出的不可恢复假设。
