SP100 诊断实验计划：先诊断再改模型

当前结论： V3 SP100 主实验中，$C_{\text{manual}}$ 没有通过成功标准： $$ CVaR(C_{\text{manual}})=2.92\% > CVaR(A)=2.86\%. $$ 但是 $B_{\text{fixed}}(\kappa=2)$ 表现较好： $$ CVaR(B_{\text{fixed}})=2.66\%. $$ 说明问题不一定是 RCVaR 风险层失效，而更可能是 手工状态映射 $\kappa(s)$ 在 SP100 上失效。

1. 诊断实验一：$\kappa(s)$ 时间序列诊断

1.1 为什么做？

$C_{\text{manual}}$ 的核心是 $\kappa(s)$。如果 $\kappa(s)$ 在危机期不上升，或者正常时期长期过高，那么它就不是有效的 market stress mapping。

核心问题： $$ \boxed{ \kappa(s_t)\text{ 是否真的在高波动、高回撤、危机阶段升高？} } $$

1.2 输入数据

robust_cvar_portfolio/outputs/v3/sp100/kappa_series.csv
robust_cvar_portfolio/outputs/v3/sp100/state_matrix.npy
robust_cvar_portfolio/outputs/v3/sp100/rolling_results.csv

1.3 需要计算什么？

指标 公式 / 逻辑 作用
$\bar{\kappa}$ 所有调仓日 $\kappa_t$ 的均值 判断平均鲁棒程度是否过高。
$\min \kappa_t,\max \kappa_t$ 时间序列最小 / 最大值 判断是否长期卡在边界。
危机期均值 2020、2022 子样本均值 判断危机期 $\kappa$ 是否更高。
正常期均值 非 2020/2022 区间均值 作为危机期对照。
相关性 $corr(\kappa_t,Vol_t)$，$corr(\kappa_t,DD_t)$ 检验 $\kappa$ 是否跟市场 stress 同向。

1.4 推荐代码逻辑

def diagnose_kappa(kappa_df):
    summary = {}

    summary["kappa_mean"] = kappa_df["kappa"].mean()
    summary["kappa_min"] = kappa_df["kappa"].min()
    summary["kappa_max"] = kappa_df["kappa"].max()
    summary["kappa_std"] = kappa_df["kappa"].std()

    summary["corr_kappa_vol"] = kappa_df["kappa"].corr(kappa_df["Vol_z"])
    summary["corr_kappa_dd"] = kappa_df["kappa"].corr(kappa_df["DD_z"])
    summary["corr_kappa_mom"] = kappa_df["kappa"].corr(kappa_df["Mom_z"])
    summary["corr_kappa_corr"] = kappa_df["kappa"].corr(kappa_df["Corr_z"])

    crisis_2020 = (kappa_df["date"] >= "2020-02-01") & (kappa_df["date"] <= "2020-06-30")
    crisis_2022 = (kappa_df["date"] >= "2022-01-01") & (kappa_df["date"] <= "2022-12-31")

    summary["kappa_2020_mean"] = kappa_df.loc[crisis_2020, "kappa"].mean()
    summary["kappa_2022_mean"] = kappa_df.loc[crisis_2022, "kappa"].mean()
    summary["kappa_normal_mean"] = kappa_df.loc[~(crisis_2020 | crisis_2022), "kappa"].mean()

    return pd.Series(summary)

2. 诊断实验二：权重集中度与权重距离诊断

2.1 为什么做？

SP100 中 $C_{\text{manual}}$ 失败，可能不是因为风险层错误，而是因为它产生了更不稳定或更集中的组合权重。

核心问题： $$ \boxed{ C_{\text{manual}}\text{ 是否比 B_fixed 或 A 更集中、更不稳定？} } $$

2.2 输入数据

outputs/v3/sp100/weights_A.csv
outputs/v3/sp100/weights_B.csv
outputs/v3/sp100/weights_C_manual.csv
outputs/v3/sp100/weights_D.csv
outputs/v3/sp100/weights_HistoricalCVaR.csv

2.3 需要计算什么？

指标 公式 解释
HHI 集中度 $HHI_t=\sum_i w_{i,t}^2$ 越高表示权重越集中。
有效持仓数 $N^{eff}_t=1/HHI_t$ 越低表示组合越集中。
最大单资产权重 $\max_i w_{i,t}$ 检验是否压到少数股票。
C-B 权重距离 $\|w^C_t-w^B_t\|_1$ 检验 C 与 fixed robust 的差异。
C-A 权重距离 $\|w^C_t-w^A_t\|_1$ 检验 C 与普通 CVaR 的差异。

2.4 推荐代码逻辑

def weight_diagnostics(weights_dict):
    records = []

    for name, W in weights_dict.items():
        hhi = (W ** 2).sum(axis=1)
        neff = 1.0 / hhi
        maxw = W.max(axis=1)

        tmp = pd.DataFrame({
            "method": name,
            "date": W.index,
            "hhi": hhi.values,
            "effective_n": neff.values,
            "max_weight": maxw.values
        })
        records.append(tmp)

    return pd.concat(records, ignore_index=True)

def pairwise_weight_distance(W1, W2):
    common_dates = W1.index.intersection(W2.index)
    common_cols = W1.columns.intersection(W2.columns)
    return (W1.loc[common_dates, common_cols] - W2.loc[common_dates, common_cols]).abs().sum(axis=1)

3. 诊断实验三：换手率与交易成本诊断

3.1 为什么做？

$C_{\text{manual}}$ 的动态 $\kappa(s)$ 可能导致每月权重跳动更大，从而增加交易成本或样本外不稳定性。

核心问题： $$ \boxed{ C_{\text{manual}}\text{ 是否因为换手率更高而损害样本外表现？} } $$

3.2 需要计算什么？

指标 公式 解释
单期换手率 $Turn_t=\|w_t-w_{t-1}\|_1$ 每次调仓变化幅度。
平均换手率 $\bar{Turn}$ 整体交易强度。
累计换手率 $\sum_t Turn_t$ 总交易强度。
交易成本贡献 $c\cdot Turn_t$ 成本是否拖累结果。

3.3 推荐代码逻辑

def compute_turnover(W):
    turnover = W.diff().abs().sum(axis=1)
    turnover.iloc[0] = W.iloc[0].abs().sum()
    return turnover

def turnover_diagnostics(weights_dict, cost_rate=0.001):
    records = []
    for name, W in weights_dict.items():
        turn = compute_turnover(W)
        tmp = pd.DataFrame({
            "method": name,
            "date": turn.index,
            "turnover": turn.values,
            "cost": cost_rate * turn.values
        })
        records.append(tmp)
    return pd.concat(records, ignore_index=True)

4.1 为什么做？

SP100 是股票池，不是 ETF。股票池的风险很大一部分来自行业集中。如果 C 在某些时段过度暴露到单一 sector，可能导致 CVaR 变差。

4.3 行业暴露公式

$$ Exposure_{g,t}^{(m)} = \sum_{i\in g} w_{i,t}^{(m)}. $$

4.4 推荐代码逻辑

def sector_exposure(W, universe):
    sectors = universe.set_index("ticker")["sector"]
    common = W.columns.intersection(sectors.index)

    W = W[common]
    sectors = sectors.loc[common]

    exposures = {}
    for sector in sectors.unique():
        tickers = sectors[sectors == sector].index
        exposures[sector] = W[tickers].sum(axis=1)

    return pd.DataFrame(exposures, index=W.index)

def sector_diagnostics(W, universe):
    E = sector_exposure(W, universe)
    sector_hhi = (E ** 2).sum(axis=1)
    max_sector = E.max(axis=1)
    return E, pd.DataFrame({
        "sector_hhi": sector_hhi,
        "max_sector_exposure": max_sector
    })

9. 诊断完成后的模型修改决策树

诊断结果 说明 后续改法
$\kappa$ 响应方向错误 状态变量或 $\beta$ 符号不合理 重新定义 $s_t$，用 validation 选择 $\beta$
$\kappa$ 过度跳动 风险度量不稳定 加入 $\kappa_t=\rho\kappa_{t-1}+(1-\rho)\kappa(s_t)$ 平滑
$\kappa_{\max}$ 小时明显改善 原模型过度鲁棒 降低 $\kappa_{\max}$，用 validation 固定

所有诊断均不支持 C state-dependent $\kappa(s)$ 在 SP100 暂不适用 论文中如实写 limitation，以 B_fixed / Historical CVaR 为强基线

10. 建议执行顺序

顺序 实验 预计耗时 原因

6 窗口长度 sensitivity 较慢 需要重跑多个窗口。

11. 当前阶段的论文写法建议

在诊断完成前，不要写：
“state-dependent $\kappa(s)$ 在所有市场上稳定优于 baseline。”

应暂时写：
“state-dependent $\kappa(s)$ 在 ETF10 和 SP30 中有效，但在 SP100 中未能稳定优于 fixed robust 或 historical CVaR baseline。本文进一步通过诊断实验分析其高维失效机制。”

12. 诊断实验完成后应形成的结论模板

12.1 如果发现 $\kappa$ 过度鲁棒

在高维 SP100 股票池中，手工 $\kappa(s)$ 的动态范围过大，使 RCVaR 过度强调少数历史尾部样本，从而放大 tail estimation noise。降低 $\kappa_{\max}$ 或平滑 $\kappa_t$ 是下一步改进方向。

12.2 如果发现权重/行业集中

状态依赖风险度量本身虽然增强了尾部保守性，但在高维个股池中可能诱导组合集中，尤其是行业集中。因此 future version 需要加入 sector-aware constraints 或 diversification regularization。

12.3 如果发现 Historical CVaR 稳定占优

Historical CVaR 在高维股票池中仍是非常强的 baseline，说明简单、稳定的尾部风险最小化在有限样本中可能优于更复杂的状态依赖风险包络。

13. 最终总结

诊断结论（2026-07-01）

$C_{\text{manual}}$ 在 SP100 失败的主因不是 κ(s) 映射方向错误，而是：

  1. κ_max=1.0 过度鲁棒 — 降至 0.5 后 CVaR 从 2.92%→2.68%，接近 B_fixed；
  2. 动态 κ 导致换手翻倍（0.61 vs A 0.27）— 样本外稳定性受损；
  3. 权重更集中（N^eff 70 vs A 88）— 高维个股池 diversification 不足。

Historical CVaR（2.46%）仍是最强 baseline；RCVaR 族内 B_fixed（2.66%）优于默认 C_manual。

下一步改模型（V5 方向）： κ_max 在 val 上选 0.25–0.75；κ 时间平滑；换手/HHI 正则；而非放弃 state-dependent κ。

本诊断计划的目标不是马上让 C_manual 变好，而是先弄清楚：

$$ \boxed{ C_{\text{manual}}\text{ 在 SP100 上失败的机制是什么？} } $$
只有明确失败原因后，才能决定是否：
1. 降低 $\kappa_{\max}$；
2. 平滑 $\kappa_t$；
3. 增加窗口长度；
4. 加入 sector cap；
5. 改成 asset-aware / sector-aware $\kappa$；
6. 或者在论文中如实把 SP100 写成 high-dimensional limitation。
